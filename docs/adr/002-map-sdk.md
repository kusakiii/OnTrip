# ADR-002：地图 SDK 选择 — 腾讯地图（仅标注）+ 高德 API（数据）

**状态**：已接受
**日期**：2026-07-02

## 背景

行程需要地图可视化（标注景点位置）和 POI 数据（评分/照片/价格）。

- 微信小程序内置 `map` 组件底层是腾讯地图
- 腾讯地图 POI API 不返回评分、照片、人均消费
- 高德地图 POI API（extensions=all）返回完整的评分、照片、价格、标签

## 决策

**地图渲染**用腾讯地图（小程序原生），**POI 数据**用高德 API。

## 理由

- 小程序 `map` 组件零额外集成成本
- 高德 API 返回的 rich POI 数据恰好满足 F7 富化需求
- 个人开发者日均 5000 次调用，TripFlow 用量远低于此

## 后果

- ✅ 地图渲染零成本、零延迟
- ✅ POI 数据丰富度满足卡片详情需求
- ❌ 两套 API Key 需要分别管理
- ❌ 高德照片 URL 在微信小程序 `<image>` 组件中的加载兼容性待验证
- ❌ V1.0 不做路线连线（静态标注版），完整路线留给 V2.0
