# ADR-001：后端平台选择 — 微信云开发（CloudBase）

**状态**：已接受
**日期**：2026-07-02
**决策者**：架构师

## 背景

TripFlow 是个人 AI 旅行规划工具，需要后端来代理 AI API 调用（隐藏 Key）、存储行程数据、代理地图 POI 请求。

候选方案：

- A: 微信云开发（CloudBase）
- B: 腾讯云轻量服务器（需备案）
- C: Cloudflare Workers（国内慢）

## 决策

选择 **微信云开发（CloudBase）**。

## 理由

- 零运维、零备案、免费额度充足
- `wx.cloud.callFunction` 原生集成，不需要额外配置 HTTPS/域名
- 云数据库（NoSQL）天然适合存储行程 JSON 文档
- 个人开发者在 V1.0 阶段数据量极小，无配额压力

## 后果

- ✅ 降低 V1.0 的运维成本和上线门槛
- ✅ 与微信小程序原生集成，开发效率最高
- ❌ 云函数冷启动 100-500ms，但 AI 生成场景下占比 <2%，无感知
- ❌ 未来若 DAU 增长或需要关系型查询，需迁移到独立服务器
- ❌ 云函数 60s 超时限制，需要 POI 查询改为并行化

## 替代方案

如 V2.0 需要更强后端能力，候选迁移路径：

- Supabase（PostgreSQL + 边缘函数）
- 阿里云 ECS + NestJS + Redis + MySQL
